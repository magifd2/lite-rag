# 依存関係レジスター

RULES.md §18 の要件に従い、すべてのサードパーティ依存関係をここに記録します。

---

## 直接依存

### `github.com/marcboeker/go-duckdb/v2` @ v2.4.3

- **目的:** DuckDB 向け CGo ベースの Go ドライバー（v2）。`database/sql` 互換性を提供し、
  `[]float32` を `FLOAT[]` カラムに SQL パラメーターとして直接渡せるため、
  `list_cosine_similarity()` による DB 側でのコサイン類似度計算が可能。
- **内製しない理由:** DuckDB ワイヤープロトコルドライバーをゼロから実装するのは現実的でない。
  DuckDB は組み込み型（別プロセス不要）であり、配列演算関数を内蔵している点で採用。
- **v1 ではなく v2 を使う理由:** go-duckdb v1 は `[]float32` を `FLOAT[]` カラムへの SQL
  パラメーターとして渡せず、埋め込みベクトルを `BLOB` として保存して Go 側でコサイン類似度を
  計算する必要があった。v2 ではこの制限が解消され、`list_cosine_similarity` による
  DuckDB 内での類似度検索が可能になった。
- **ライセンス:** MIT
- **コンプライアンス:** 社内・商用利用ともに制限なし。

### `github.com/spf13/cobra` @ v1.10.2

- **目的:** サブコマンド構造、フラグ解析、ヘルプテキスト生成を提供する CLI フレームワーク。
- **内製しない理由:** 標準の `flag` パッケージはサブコマンドをサポートしない。
  Cobra は Go CLI のデファクトスタンダードであり、ボイラープレートを大幅に削減できる。
- **ライセンス:** Apache 2.0
- **コンプライアンス:** 社内・商用利用ともに制限なし。

### `github.com/BurntSushi/toml` @ v1.6.0

- **目的:** TOML 設定ファイルの解析。
- **内製しない理由:** TOML 解析の実装は複雑。本ライブラリは依存関係なし・軽量・活発にメンテされている。
- **ライセンス:** MIT
- **コンプライアンス:** 制限なし。

### `golang.org/x/text` @ v0.35.0

- **目的:** 日英混在テキストの Unicode 正規化（NFKC）。`internal/normalizer` で使用し、
  全角文字・半角カタカナ・互換文字をチャンク分割・埋め込み前に正規化する。
- **内製しない理由:** Unicode 正規化アルゴリズムは複雑で誤実装が起きやすい。
  Go チームがメンテする標準ライブラリ拡張を使用するのが適切。
- **ライセンス:** BSD 3-Clause
- **コンプライアンス:** 制限なし。
